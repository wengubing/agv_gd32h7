# agv_gd32h7 仓库说明

# 开发任务
将stm32f4主控AGV软件系统移植到另一个gd32h7控制器上，并且要参考另一个stm32h7系统的某些架构设计。

## 方案一
前人工，1.买gd32的开发板，跑通一些简单的例程；2.参考stm32h7的rtx5例程移植操作系统；3.根据目标控制器硬件接口设计和开发板级api；4.将原来已经分层设计的stm32f4系统的应用层移植到新的gd32h7上调试测试；
## 方案二
ai辅助移植，1.人工大致将系统框架搭建起来；2.逐个调试接口和功能，过程中遇到问题通过ai分析和自动修改代码，再人工调试测试；
## 方案三
ai主导移植，ai完全自主参考已有工程和设计约束进行移植，自动修改和测试，直至完成，并形成开发过程文档，最终交付人工再测试；

## 1. 仓库定位
本仓库用于集中管理 AGV 相关控制器资料与工程代码，当前内容由多套历史/现行平台组成，覆盖：
- STM32H7（安富莱示例与模板工程）
- STM32F4（航发旧 AGV 系统工程）
- GD32H7（野火开发板例程与航发新控制器设计资料）

## 4. 各主目录快速导航

### 4.1 安富莱stm32h7
包含三套主工程：
- `RTX5+RL-USB+FL-FlashFS+RL-TCPnet+emWin6.x(V1.1)/`：RTX5 + USB + FlashFS + TCPnet + emWin 综合示例。
- `V7-999_出厂程序（V1.0e）/`：出厂程序，目录结构含 `Doc/`、`Libraries/`、`Project/`、`User/`。
- `V7-RTX5 Template/`：RTX5 工程模板，适合作为新功能移植起点。

### 4.2 航发旧agv系统 stm32f4
- 主工程位于 `AGV程序/`。
- 关键目录包括：`ACC_Project/`、`app_SkeletonAGV/`、`CBB/`、`HAL/`、`HMI/`、`Drivers/`、`Project/`、`Xdoc/` 等。
- `Readme.md` 已给出该工程模块划分与系统架构图入口。

### 4.3 航发新gd32h7控制器硬件原理与软件移植
- 当前以设计资料为主（例如需求说明、资源分配、硬件原理 PDF、分工文档）。
- 适合作为新控制器软件开发和硬件联调的输入文档区。

### 4.4 野火gd32h7
- 主目录：`ebf_gd32h7_Development_Board_factory_test_program/`。
- 关键子目录：`App/`、`Firmware/`、`Project/`、`User/`、`Doc/`、`lwip-2.1.2/`。
- 可作为 GD32H7 外设/协议栈移植参考基线。
- 文件夹GD32_CAN为开发板上跑通的can收发工程
- 文件夹H7_Demo为移植了RTX5和跑通can收发的工程

## 5. 后续维护建议
1. 每个主目录补一份最小 `README.md`：说明芯片型号、工具链版本、入口工程文件。

3. 对体积较大的第三方库目录（如 `lwip-2.1.2`）在说明中标注来源版本与改动策略。
4. 如需持续追踪导入节奏，建议按“平台名+日期”规范提交信息。
